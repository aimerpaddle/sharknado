How We Did It

Using GitHub pages, we are able to create a website quickly and efficiently. The steps to creating a GitHub webpage are simple and straightforward:

Step 1:

Go to the GitHub pages webpage, scroll down slightly until you see two buttons that say User or organization site and Project site. Select Project site.

Step 2:

The page will have several steps to show you how to create your very own GitHub webpage. If you follow the steps on the website you should have no problem creating a GitHub page.

Step 3:

The last step is to add content by going to your gh-pages branch in your repository and editing the source filles as you see fit.

*Note:

Creating a GitHub page for a repository will create a branch called gh-pages. So, if you want to work on your Github pages, make sure that you are pulling from and pushing to the gh-pages branch. Some helpful commands to know would be:

git clone –b <branchname> <URL>

This allows you to get the code from a branch in the repository.

git push origin HEAD:<branchname>

Pushes changes to a specific branch in the repository.

If you are working in a team with multiple people working on the same files it will be useful to be able to sync your fork. Syncing your fork allows you to get the current version of the website so that you are not accidentally working on an older version of the website. GitHub provides a help page about how to sync a fork that is very helpful.
